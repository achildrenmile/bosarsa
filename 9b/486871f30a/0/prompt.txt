# Plan: Migrate BOS-ARSA to Eleventy (11ty)

## Context

The BOS-ARSA website has 4 HTML pages with massive code duplication — nav, footer, CSS variables, font-face declarations, favicons, and JS are copy-pasted across every file. Adding a new page means copying 150+ lines of boilerplate. The press page has 6 hand-coded article cards that will grow over time. SEO files (robots.txt, sitemap.xml, llms.txt) are also manually maintained.

**Goal:** Migrate to Eleventy (11ty) — a zero-config static site generator that outputs plain HTML. The nginx/Docker deployment stays identical, we just add a build step.

## Why Eleventy

- Zero-config, minimal dependencies (~1 npm package)
- Nunjucks templating: layouts, partials, loops, data files
- Output is plain static HTML — served by nginx exactly as before
- JSON data files for press articles (add article = add JSON entry)
- Can generate sitemap.xml, robots.txt, llms.txt from templates
- No client-side framework — same HTML output as today

## New Project Structure

```
bosarsa/
├── src/
│   ├── _includes/
│   │   ├── base.njk              # Full HTML document shell
│   │   ├── nav.njk               # Full navigation (logo, links, hamburger)
│   │   └── footer.njk            # Full 3-column footer
│   ├── _data/
│   │   ├── site.json             # Global: name, url, email, address, ZVR, etc.
│   │   └── presse.json           # Press articles array
│   ├── css/
│   │   └── base.css              # All shared CSS (vars, fonts, nav, footer, responsive)
│   ├── index.njk                 # Homepage
│   ├── presse.njk                # Press page (loops over presse.json)
│   ├── impressum.njk             # Impressum (content only, layout from base)
│   ├── datenschutz.njk           # Datenschutz (content only, layout from base)
│   ├── robots.njk                # -> robots.txt
│   ├── sitemap.njk               # -> sitemap.xml (auto-lists all pages)
│   └── llms.njk                  # -> llms.txt (pulls from site.json)
├── fonts/                         # Stays in root (passthrough copy)
├── favicon/                       # Stays in root (passthrough copy)
├── *.mp4, *.png, *.webp          # Static assets (passthrough copy)
├── .eleventy.js                   # Eleventy config

├── Dockerfile                     # Multi-stage: node build -> nginx serve
├── docker-compose.yml             # Unchanged
├── deploy-production.sh           # Unchanged
└── nginx.conf                     # Unchanged
```

### 3. `src/_includes/base.njk` — Single layout for all pages
- `<head>` with fonts, CSS, favicons, SEO meta (title, description, canonical, OG, Twitter)
- Includes `nav.njk` and `footer.njk`
- Content block for page-specific content
- Shared JS at bottom (lucide, year, hamburger, scroll)
- Front matter controls: `title`, `description`, `canonical`, `activeNav`, `pageCSS`

### 4. `src/presse.njk` — Loop replaces 6 copy-pasted cards
```nunjucks
{% for article in presse %}
  <a class="article" href="{{ article.url }}" target="_blank" rel="noopener noreferrer">
    <div class="article__date">{{ article.date }}
      <span class="article__tag article__tag--{{ article.tag }}">{{ article.tagLabel }}</span>
    </div>
    <h2 class="article__title">{{ article.title }}</h2>
    <div class="article__source">{{ article.source }}</div>
    <p class="article__desc">{{ article.desc }}</p>
  </a>
{% endfor %}
```

### 5. SEO files — Generated from data
- **sitemap.njk**: Loops over 11ty's `collections.all` to auto-list all pages
- **robots.njk**: Simple template referencing `site.url`
- **llms.njk**: Pulls org info from `site.json`, board members, etc.

### 6. `Dockerfile` — Multi-stage build
```dockerfile
FROM node:20-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm ci --production
COPY . .
RUN npx @11ty/eleventy

FROM nginx:alpine
RUN rm /etc/nginx/conf.d/default.conf
COPY nginx.conf /etc/nginx/conf.d/bosarsa.conf
COPY --from=build /app/_site/ /usr/share/nginx/html/
EXPOSE 80
```

### 7. `.eleventy.js` — Config
- Input: `src/`, Output: `_site/`
- Passthrough copy: `fonts/`, `favicon/`, `*.mp4`, `*.png`, `*.webp`, `nginx.conf`
- Nunjucks as template engine
